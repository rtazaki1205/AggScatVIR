Python package: aggscatpy

aggscatpy is a python package that helps to analyze the data contained in this database. The main functionalities are as follows.

  • view each particle shape,

  • check and plot the optical properties of dust particles,

  • convert the definition of the scattering matrix from RADMC-3D’s one to Bohren & Huffman’s one,

  • perform size distribution averaging of the optical properties,

  • output files in various formats.

Suppose you have already installed the package via pip, you can import the package:

[1]:

import aggscatpy

View particle shape

The view_particle function displays a particle shape used to compute the optical properties. For example,

[2]:

aggscatpy.view_particle(partype='CALP',size='4096',amon='100nm')

where partype is the particle type, size is the size of the aggregate (in units of the number of monomers), and amon is the monomer radius. The arguments that are used to call the function obey the same rules as the dust file name. By default, the command results in showing all realizations of each particle model. If you want to display only one of the realizations, add ireal in the argument (note: the realization index starts from 1, not 0):

[3]:

aggscatpy.view_particle(partype='CALP',size='4096',amon='100nm',ireal='1')

We can get a list of a possible combination between the amon and size arguments by using the get_sizelist function:

[4]:

aggscatpy.get_sizelist('CALP')

 Available amon and size for CALP
 amon='100nm' | size='8','16','32','64','128','256','512','1024','2048','4096'
 amon='200nm' | size='8','16','32','64','128','256','512'
 amon='400nm' | size='8','16','32','64'

The command to display irregular grains is

[5]:

aggscatpy.view_particle(partype='grs',size='1_6000')

In this case, partype has to be 'grs'. The size argument represents the volume-equivalent radius of grains in units of microns (the decimal point needs to be replaced by the underscore). For example, size='1_6000' means that the volume-equivalent radius is \(1.6~\mu\)m. You can get a list of the available size arguments for irregular grains:

[6]:

Reading each dust model file

Basic formula

To read one of dustkapscatmat_XXX.inp stored in the AggScatVIR database, we can use the dustmodel class.

The basic command for reading an aggregate model is

[7]:

a=aggscatpy.dustmodel(partype='CALP',size='4096',amon='100nm',comp='amc')

where partype is the particle type, size is the size of the largest aggregate in the size distribution (in units of the number of monomers), amon is the monomer radius, and comp is the monomer composition. After running this command, the instance a will contain the physical and optical properties of the dust model of CALP_Nmax4096_100nm_amc.

The command for reading an irregular grain model is

[8]:

b=aggscatpy.dustmodel(partype='grs',size='1_6000',comp='org')

The size argument specifies the maximum volume-equivalent radius of grains.

By default, the dustmodel class reads a dust file with size distribution and no forward-scattering chopping. You can change these default settings by adding the dist and/or chop option(s) when calling the class. For example, if you want to switch off the size distribution, set dist=False (default: dist=True). If you want to have forward scattering chopping, set either chop='chop5' or chop='chop10' (default: chop='nochop'). chop5 and chop10 refer to cases where the forward scattering peak is chopped at scattering angles below 5 or 10 degrees, respectively. The exemplary command for these cases are

[9]:

a2=aggscatpy.dustmodel(partype='CALP',size='4096',amon='100nm',comp='amc',dist=False,chop='chop5')

We can make some plots for the optical properties of the dust particle by using the showmodel function:

[11]:

aggscatpy.showmodel(a)

The above plots summarize the absorption and scattering opacities (\(\kappa_\mathrm{abs}\), \(\kappa_\mathrm{sca}\)) (left top), the single scattering albedo (\(\omega\)), the asymmetry parameter (\(g\)), and the maximum degree of linear polarization (\(P_\mathrm{max}\)) (left bottom), the scattering phase function (right top) and the degree of linear polarization (right bottom) at various wavelengths.

The showmodel function has some options that might be useful for you: (1) If you set opacity=False, the left top panel will be replaced by cross sections (\(\mu\mathrm{m}^2\)) instead of the opacity (\(\mathrm{cm}^2/\mathrm{g}\)) (2) If you set savefig=True, the plot will be saved. The file name will be generated automatically, but you can set your own file name by adding fn=your_favorate_filename.

Scattering matrix in Bohren & Huffman’s definition

It is sometimes useful to have the scattering matrix elements in the Bohren & Huffman definition. You can switch the definition into Bohren & Huffman’s one by setting smatBH=True when calling the dustmodel class:

[17]:

abh=aggscatpy.dustmodel(partype='CALP',size='4096',amon='100nm',comp='amc',smatBH=True)

All the values in the attributes (smat11,smat12,smat22,smat33,smat34,smat44) have been replaced by the ones in Bohren & Huffman’s definition (so smat11 now contains \(S_{11}\) instead of \(Z_{11}\)). Thus \(S_{11}(\lambda,\theta)\) looks like this:

Perform size distribution averaging and apply forward scattering chopping

This database contains the optical properties averaged over a particle-size distribution. The size distribution is assumed to have a simple power-law form:

\[n(a_\mathrm{V})da_\mathrm{V}\propto a_\mathrm{V}^{-q}da_\mathrm{V}~(a_\mathrm{min}\le a_\mathrm{V} \le a_\mathrm{max}),\]

where \(a_\mathrm{V}\) is the volume-equivalent radius and \(n(a_\mathrm{V})da_\mathrm{V}\) is the number density of particles within a radius range of \([a_\mathrm{V},a_\mathrm{V}+da_\mathrm{V}]\). For aggregates,

\[\begin{split}a_\mathrm{min}=a_\mathrm{mon}N_\mathrm{min}^{1/3},\\ a_\mathrm{max}=a_\mathrm{mon}N_\mathrm{max}^{1/3},\end{split}\]

where \(N_\mathrm{min}\) and \(N_\mathrm{max}\) are the number of monomers of the smallest and largest aggregate in the size distribution, respectively. By default, we assume \(q=3.5\), \(N_\mathrm{min}=8\) for aggregates, and \(a_\mathrm{V,min}=0.2~\mu\)m for irregular grains.

However, you may be interested in the optical properties for different size distributions, such as \(q\ne3.5\) . By using distaverage, we can perform size distribution averaging of the optical properties with user-specified values of \(a_\mathrm{min}\), \(a_\mathrm{max}\), and \(q\). In addition, it can also be used to apply forward scattering chopping with a user-specified chop angle.

Basic formula

The basic command is

[19]:

d=aggscatpy.distaverage(partype='CALP',sizemin='8',sizemax='4096',comp='amc',amon='100nm',powind=3.5)

The newly created instance d contains the distribution-averaged optical properties. The sizemin and sizemax represent the smallest and largest aggregates in the distribution in units of the number of monomers, respectively, and powind is the power-law index \(q\).

The command for irregular grains is

[20]:

e=aggscatpy.distaverage(partype='grs',sizemin='0_2000',sizemax='1_6000',comp='org', powind=3.5)

As with the case of dustmodel, an instance created by distaverage can be substituted into functions: dustspec(), showmodel(), and write_data() (the last function will be introduced later).

Since the parameters used above are exactly the same as the default size distribution, the results have to agree with those contained in the default dust files. As shown below, these two are in perfect agreement.

Examples

If you want the size distribution between \(N_\mathrm{min}=128\) to \(N_\mathrm{max}=2048\) with the default power-law index (\(q=3.5\)):

[22]:

d=aggscatpy.distaverage(partype='CALP',sizemin='128',sizemax='2048',comp='amc',amon='100nm')

… and change the power-law index to \(q=2.5\):

[23]:

d=aggscatpy.distaverage(partype='CALP',sizemin='128',sizemax='2048',\
              comp='amc',amon='100nm',powind=2.5)

… and change the chop angle to 7 degrees:

[24]:

d=aggscatpy.distaverage(partype='CALP',sizemin='128',sizemax='2048',\
              comp='amc',amon='100nm',powind=2.5,chopang=7)

If you set \(N_\mathrm{min}=N_\mathrm{max}\), the results will be the same as the single-sized case.

You can also change the definition of the scattering matrix:

[25]:

dbh=aggscatpy.distaverage(partype='CALP',sizemin='128',sizemax='2048',comp='amc',amon='100nm', smatBH=True)

Writing files

Once you have generated the optical properties with a new particle-size distribution and/or chop angle, you may wish to save them as a RADMC-3D readable format. To do this, you can use the write_data function. An instance created by either the dustmodel or distaverage class can be passed to the write_data function to write a new file.

Suppose you have already created an instance d, then you can type:

[26]:

aggscatpy.write_data(d)

writing ... dustkapscatmat_CALP_Nmin128_Nmax2048_pow2.5_100nm_amc_chop7.0.inp

The filename will be automatically generated based on the minimum and maximum particle sizes, power-law index, and chopping angle (if non-zero). You can also modify the filename by setting the fn option if you want.

Without any options, the generated file will have a standard ‘dustkapscatmat_XXX’ format.

Important

The definition of the scattering matrix to be written in the file will be the same as the one defined in each instance. Since the above instance d has the scattering matrix with the RADMC definition, the generated file contains \(Z_{ij}\). In this case, the generated file is directly readable by RADMC-3D.

Conversely, if you have created an instance that has the scattering matrix in Bohren & Huffman’s definition, the following will happen:

[27]:

aggscatpy.write_data(dbh)

writing ... dustkapscatmat_CALP_Nmin128_Nmax2048_pow3.5_100nm_amc.out

In this case, the structure of the generated file is the same, but the file contains \(S_{ij}\) instead of \(Z_{ij}\). To avoid confusion, the file extension has been changed from .inp to .out so that the file is not readable by RADMC-3D anymore. You can also check the definition in the header part of each generated file.

If you are only interested in the angle-integrated quantities (opacity, asymmetry parameter, …), you can add output='dustkappa':

[28]:

aggscatpy.write_data(a,output='dustkappa')

writing ... dustkappa_CALP_Nmax4096_100nm_amc.out

The generated file now only contains the angle-integrated quantities and not scattering matrix elements. If you are interested in cross sections instead of opacity, you can add the following option so that the output file will contain cross sections (\(\mu\mathrm{m}^2\)):

[29]:

aggscatpy.write_data(a,output='dustkappa', opacity=False)

writing ... dustkappa_CALP_Nmax4096_100nm_amc.out

Conversely, if you are only interested in scattering matrix elements, add output='smat':

[30]:

aggscatpy.write_data(a,output='smat')

Comparing phase functions of different fractal dimensions

For example, let’s investigate how the total-intensity phase function of aggregates changes with fractal dimension at the \(H\) band wavelength (\(\lambda=1.63~\mu\)m).

To do so, first create some lists:

[33]:

pt=['FA19','FA15', 'FA13','FA11'] # a list of particle types
nmono=['2048','512', '256','128']      # a list of the number of monomers
label=['$D_f=1.9$','$D_f=1.5$','$D_f=1.3$','$D_f=1.1$'] # a list of labels for lenged

where we have assumed that a smaller number of monomers for a lower fractal dimension to approximately fix the aggregate radius. To check the characteristic radius of an aggregate, you can use the get_radius_and_porosity function

[34]:

for i in range(len(pt)):
    ac,por=aggscatpy.get_radius_and_porosity(pt[i],'100nm',nmono[i])
    print('Particle Type: %4s'%pt[i],' Number of monomers = %4s'%nmono[i],' agg radius = %5.2f um'%ac)

Polarization phase functions

Let’s give you one more example. In this example, we will see how the polarization phase function depends on aggregate radius and structure. Similar to the above example, we first make some lists:

nmono=['8','16','32','64','128','256','512','1024','2048'] # a list of the number of monomers
label=['N=8','16','32','64','128','256','512','1024','2048'] # a list of labels for lenged

Then, read and plot the dust models iteratively in the loop:

[37]:

iwav=4 # to fix wavelength at 1.630 um
iangn=90 # a scattering angle used for normalization

fig, (ax1,ax2) =plt.subplots(1,2,figsize=(14,5))
aggscatpy.plots.set_ang_vs_phase(ax1)
aggscatpy.plots.set_ang_vs_phase(ax2)

ax1.set_title('Compact aggregates: CAHP',fontsize=20)
ax2.set_ylabel('Polarized phase function',fontsize=20)
ax2.set_ylim(2.0e-1,10)
ax2.set_title('Fractal aggregates: FA19',fontsize=20)
for i in range(len(nmono)):
